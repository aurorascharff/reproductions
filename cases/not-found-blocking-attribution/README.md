# Case: `/_not-found` blocking error doesn't name the root layout

Next.js **16.3 canary** with `cacheComponents: true`. The **root layout** reads
`cookies()` outside `<Suspense>`, which makes every route blocking. But
`next build` fails on the **synthetic `/_not-found` route** — a route with **no
user file** — and the error never mentions the root layout that actually caused
it.

```bash
cd cases/not-found-blocking-attribution
npm install
npm run build
```

## Why this happens

`/_not-found` is generated by Next.js; there is no `app/_not-found/page.tsx`. But
it still renders **through the root layout** (`app/layout.tsx`), like every other
route. When the root layout calls `cookies()` outside `<Suspense>`, the layout
becomes blocking, so prerendering `/_not-found` fails.

`next build` also **stops at the first blocking route** it hits, and
`/_not-found` is processed early — so this confusing synthetic-route error is
often the **first** thing you see after enabling Cache Components.

## What we expected

The build error to name the originating user segment — the root layout's
`cookies()` call (file + line) — so the fix target is obvious.

## What you actually see

```
Error: Route "/_not-found": Next.js encountered uncached or runtime data during prerendering.
  ...
Export encountered an error on /_not-found/page: /_not-found, exiting the build.
```

No mention of `app/layout.tsx`. A developer (or agent) looks for an
`app/_not-found` file to fix, finds none, and may try to add
`export const instant = false` to a route that has no file — which does nothing.

## Fix

Wrap the request-data read in `<Suspense>` (extract it into an async child), or
opt the **root layout** out with `export const instant = false`. The point of
the repro is that the error should point you here without needing to know that
`/_not-found` renders through the root layout.

```tsx
// app/layout.tsx
import { Suspense } from 'react'

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <Suspense fallback={<div />}>
          <ThemedShell>{children}</ThemedShell>
        </Suspense>
      </body>
    </html>
  )
}
```
